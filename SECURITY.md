
# Security Policy

## Supported Versions

| Version | Supported |
| -------- | --------- |
| Latest   | ✅ |
| Older Releases | ❌ |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately.

Do **not** create a public GitHub issue for security vulnerabilities.

Instead, contact the project maintainer:

**Email:** nancypatel016@gmail.com

Include the following information:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested mitigation (if known)

---

## Response Process

After receiving a report, we aim to:

- Acknowledge the report within 72 hours.
- Investigate the issue promptly.
- Develop and test a fix.
- Release a security update when appropriate.
- Credit the reporter (if they wish).

---

## Scope

This policy applies to:

- Authentication
- Authorization
- API endpoints
- Database interactions
- Dependencies
- Infrastructure
- Third-party integrations

---

Thank you for helping keep DevLink secure.
